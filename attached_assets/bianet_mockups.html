Seed — buyer home screen
The entry point for buyers. Personalized greeting, search bar, category grid, promotional banner, and featured listings. Designed for low-end Android — no heavy animations, skeleton loaders on images, offline cache for last-loaded listings.
Search bar
Full-text search via PostgreSQL tsvector. Instant results with debounced query.
Category grid
6 main categories seeded by admin. Tap navigates to filtered browse with pre-applied category filter.
Featured listings
Paid featured_until listings from verified vendors. Commission model in Phase 2.

Seed — listing detail
Full product detail with photo gallery, vendor profile, verified badge, quantity stepper (respects MOQ), and total price calculation in real-time. Direct message button opens order-scoped chat with the vendor.
Verified vendor badge
Only shown when kyc_status = verified AND approval_status = approved in DB.
Quantity stepper
Min enforced as listing.moq. Max enforced as listing.available_qty. Total = qty × unit_price.
"Add to cart" flow
Snapshots unit_price at add time into cart.items jsonb. Price changes don't affect existing cart.

Seed — vendor dashboard
Role-switched view of the same Seed app. After BVN/NIN verification and admin approval, the user's role = vendor and this dashboard appears. Earnings, pending orders, and wallet balance are the first things a vendor sees every morning.
Earnings cards
Aggregated from commission_ledger + vendor_wallets. Today = where created_at date = today.
Pending orders badge
Orders with status = paid where vendor hasn't confirmed. Amber highlight — requires action.
Wallet section
available_balance (cleared for withdrawal) vs pending_balance (still in escrow).
